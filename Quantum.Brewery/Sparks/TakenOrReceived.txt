🎯 Bob’s Answer — DP Question: Handling const char** Arrays in CallImpl<>

For SDK methods that use const char** arrays (e.g., entitlement IDs, tags, etc.), here are the three mandatory memory handling steps:

✅ 1. Conversion Before SDK Call
Use ProsStringArrayCast(...) to convert a TArray<FString> into const char** (C-style array):

SdkStruct->string_array = ProsStringArrayCast(UEArray);
SdkStruct->string_array_count = UEArray.Num();
✅ 2. Use of the Data During the SDK Call
Pass the converted struct as part of the SDK call.
Remember: SDK does not take ownership — it just borrows the memory.

auto Result = sdk_fn(auth, SdkStruct, cb);
✅ 3. Memory Cleanup After the Call
Always release memory allocated via ProsStringArrayCast() using ProsArrayRelease():

ProsArrayRelease(SdkStruct->string_array, SdkStruct->string_array_count);
This should be inside DestroySdkType() or the RAII wrapper destructor.

🧼 Why is this critical?

Because the SDK expects short-lived memory.

Failure to release = memory leak.

Failure to allocate properly = garbage in SDK call.

Failure to respect non-ownership = subtle bugs in long sessions.

		template<typename T>
		class FScopedShallowSdk {
			T* Ptr = nullptr;
			size_t Count = 0;

		public:
			// c/d
			template<typename InType>
			explicit FScopedShallowSdk(const TArray<InType>& In)
				: Ptr(TArrayToShallowSdk<T, InType>(In))
				, Count(static_cast<size_t>(In.Num())) 	{}
			~FScopedShallowSdk() { if (Ptr) Pros::Release::ProsRelease(Ptr); }

			// access
			T* Get() const { return Ptr; }
			size_t Num() const { return Count; }

			// non-copyable, move-only
			FScopedShallowSdk(FScopedShallowSdk&& Other) noexcept
				: Ptr(Other.Ptr), Count(Other.Count) {
				Other.Ptr = nullptr; Other.Count = 0;
			}
		private:
			FScopedShallowSdk(const FScopedShallowSdk&) = delete;
			FScopedShallowSdk& operator=(const FScopedShallowSdk&) = delete;
			FScopedShallowSdk& operator=(FScopedShallowSdk&& Other) = delete;
